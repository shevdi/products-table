## Context

Приложение products-table реализуется по TECH_SPEC.MD. Требуется план разработки в `docs/DEV_PLAN.MD` для отслеживания прогресса. План должен быть в формате чеклиста (markdown `- [ ]`).

## Goals / Non-Goals

**Goals:**
- Структурированный план по фазам
- Чеклист для каждой задачи
- Соответствие структуре и требованиям из TECH_SPEC

**Non-Goals:**
- Детальные подзадачи внутри каждой фазы (достаточно уровня «сделано / не сделано»)
- Оценки времени

## Decisions

**Формат:** Markdown с заголовками фаз и чекбоксами `- [ ]` для каждой задачи.

**Структура фаз:** Соответствует порядку из TECH_SPEC и зависимостям:
1. Подготовка (проект, инструменты)
2. Базовые компоненты (Storybook)
3. Авторизация
4. Список товаров
5. Поиск
6. Добавление товара
7. Полировка

**Расположение:** `docs/DEV_PLAN.MD` — рядом с TECH_SPEC для удобства.

## Risks / Trade-offs

- План статичен; при изменении TECH_SPEC потребуется ручное обновление DEV_PLAN.
