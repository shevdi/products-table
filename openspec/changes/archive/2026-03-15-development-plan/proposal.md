## Why

Нужен структурированный план разработки для реализации приложения products-table по [TECH_SPEC.MD](../docs/TECH_SPEC.MD). План в виде чеклиста позволит отслеживать прогресс и не упустить требования.

## What Changes

- Создать `docs/DEV_PLAN.MD` — план разработки в формате чеклиста
- План охватывает все фазы: инициализация проекта, базовые компоненты, авторизация, список товаров, поиск, добавление, полировка

## Capabilities

### New Capabilities

- `dev-plan`: Документ плана разработки с чеклистом задач по TECH_SPEC

### Modified Capabilities

- (нет)

## Impact

- Добавляется новый документ `docs/DEV_PLAN.MD`
- План служит ориентиром при реализации; не влияет на код напрямую
